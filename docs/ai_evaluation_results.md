## Methodology

1. **Dataset** — A fixed in-process dataset of 50 scenarios generated by rotating four candidate templates per archetype. No randomness; the dataset is identical on every run.
2. **Matcher** — `services/ai-agent/src/skills/job_matcher.py` is invoked directly. The same skill that runs in the Kafka shortlist pipeline is exercised here.
3. **Precision@5** — Mean fraction of the top-5 picks that are in the per-scenario ground-truth set.
4. **Approval rate** — Per-candidate decisions on the top 3 of every scenario, gated by score ≥ `0.69`.
5. **Latency** — Evaluation-modelled end-to-end latency from a deterministic distribution calibrated to production SLO targets. Not a wall-clock measurement of the in-process matcher.

Precision@5 and approval rate are computed from the matcher's actual output. Latency is from a deterministic modelled distribution (evaluation-only). Re-running this script produces the same numbers because all inputs are deterministic.
